The window opened with the v0.1 baseline already shipped — mermaid wired and seven LLM-authored skills present. Activity since has been a focused push on the v0.2 deterministic-renderer track: two new wired routes plus one structural cleanup, then a forced peer-dep migration.

#7 wired the diagram route — four fgraph topologies (sequence, layered, linear-flow, radial-hub) shipped behind type:"diagram". The renderer takes a structured object and emits HTML with computed coordinates, so the LLM no longer has to draw boxes itself.

Decision log 4 entries
Wire chart as a deterministic renderer rather than another LLM-authored skill
Charts have well-defined geometry — bars, slices, lines, axes. Spending tokens to redraw the same axis math every call is waste. A renderer takes a structured spec and emits the same HTML deterministically.
ConsideredLLM-authored chart skill (rejected: non-reproducible, expensive); leaning on Chart.js as a runtime dep (rejected: would break offline file:// playback).
PR#10
One PI tool, many types — not one tool per visual kind
Earlier sketches had separate tools per visual. That bloats PI's tool surface and forces every new visual type to ship as a new tool. A single lumen-generate_visual with a discriminator extends by adding an enum value and a renderer.
ConsideredOne tool per type (rejected: surface area + churn); a single generate with mode-detection from content shape (rejected: ambiguous, harder to validate).
SourceEstablished at v0.1 lift; reaffirmed in #7 and #10.
Promote aesthetics out of per-skill templates
v0.1.x kept palette CSS inside each skill's templates/ dir. Five copies of the editorial palette drifted within a release cycle. Moving them to skills/_shared/aesthetics/ gives one source of truth.
ConsideredInline tokens in each renderer (rejected: same drift); a single mega-stylesheet (rejected: makes per-aesthetic A/B harder).
PR#9
Follow upstream from @sinclair/typebox to unscoped typebox
Not lumen's choice. pi-coding-agent 0.70 dropped @sinclair/typebox 0.34 and adopted the unscoped typebox 1.x package — same library, republished under a new name with a major bump. Bumping pi peers required matching.
ConsideredPinning to pi 0.66 indefinitely (rejected: would diverge from active pi development).

Mental model essentials 7 invariants
01
Single-file output. Every skill emits one self-contained HTML. No CDN deps, no external assets at render time — the file plays offline via file://.
02
Deterministic vs. LLM-authored is a per-skill choice. Skills with well-defined geometry get a renderer (mermaid, diagram, chart). Skills that need compositional judgment stay LLM-authored (slides, guides, recaps, gallery, fact-check).
03
Aesthetics live in skills/_shared/aesthetics/. Never re-author palette CSS inside a skill — import from there.
04
Versioning is release-please-driven. Never edit version strings by hand — wire files into release-please-config.json and let CI bump them via x-release-please-version markers.
05
Tests run with bun run test — never bun test (bun's built-in runner ignores vitest config and picks up upstreams/).
06
The same skills/ tree ships to two channels. CC consumes it via the plugin manifest; PI consumes it via dist/skills/ after the prebuild script copies it across.
07
Peer deps use "*". We're a plugin/extension — the host pins versions. We pin in devDeps for our own builds.
